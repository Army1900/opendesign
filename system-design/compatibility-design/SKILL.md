# 兼容性设计 Skill

分析系统在 API 升级、数据协议变化、多集群、老版本 APP 等场景下的兼容性设计。

## 输出路径

```
docs/system-design/[项目名]/兼容性设计.md
```

---

## 兼容性维度

### 1. API 兼容性
| 场景 | 问题 | 解决方案 |
|------|------|----------|
| 接口升级 | 字段变更、废弃、新增 | 版本化 (/v1/, /v2/) |
| 参数变化 | 必填变可选、类型变更 | 默认值、类型转换层 |
| 响应变化 | 字段增删、结构重组 | 字段兼容层、废弃标记 |
| 行为变化 | 逻辑调整、边界变化 | Feature Flag、灰度发布 |

### 2. 数据协议兼容性
| 场景 | 问题 | 解决方案 |
|------|------|----------|
| 消息格式升级 | Topic 数据结构变化 | 版本号字段、多版本消费 |
| 事件字段变更 | 新增/删除/重命名字段 | Schema Registry、默认值 |
| 编码格式变化 | JSON ↔ Protobuf | 双格式支持、转换层 |

### 3. 数据库兼容性
| 场景 | 问题 | 解决方案 |
|------|------|----------|
| MySQL 版本 | 5.7 vs 8.0 | SQL 语法差异、索引差异 |
| PostgreSQL 版本 | 12 vs 15 | 特性差异、扩展差异 |
| 字段类型变更 | INT → BIGINT | 渐进式迁移、双写 |
| 表结构变更 | 加列/删列/改列 | 灰度迁移、兼容层 |
| 索引变更 | 新增/删除索引 | 在线 DDL、低峰期 |

### 4. 缓存兼容性
| 场景 | 问题 | 解决方案 |
|------|------|----------|
| Redis 版本 | 5.x vs 6.x vs 7.x | 命令差异、数据结构差异 |
| Key 结构变化 | 字段增删 | 版本化 Key、渐进迁移 |
| 序列化变化 | JSON ↔ Protobuf | 双格式支持 |
| 集群模式 | 单机 → 哨兵 → 集群 | 配置兼容、客户端适配 |

### 5. 内存数据兼容性
| 场景 | 问题 | 解决方案 |
|------|------|----------|
| 本地缓存 | 数据结构变化 | 版本号、渐进刷新 |
| 分布式缓存 | 一致性问题 | 版本号、双写验证 |
| 会话数据 | 结构升级 | 版本化、降级处理 |

### 6. 定时任务兼容性
| 场景 | 问题 | 解决方案 |
|------|------|----------|
| XXL-Job 版本 | 2.x → 3.x | API 差异、配置迁移 |
| 任务参数变化 | 参数新增/变更 | 默认值、参数校验 |
| 执行器版本 | 新旧执行器共存 | 版本号、灰度切换 |
| 任务结果 | 数据结构变化 | 兼容解析 |

### 7. 消息队列兼容性
| 场景 | 问题 | 解决方案 |
|------|------|----------|
| Kafka 版本 | 0.10 → 1.x → 2.x → 3.x | 客户端差异、协议差异 |
| RabbitMQ 版本 | 3.8 → 3.11 | 插件差异、配置差异 |
| Topic/Queue 结构 | 分区变化、配置变化 | 兼容配置、灰度迁移 |
| 消费者组 | 新旧消费者共存 | 多组消费、偏移管理 |

### 8. 多集群兼容性
| 场景 | 问题 | 解决方案 |
|------|------|----------|
| 数据同步 | 主从延迟、双向同步 | 同步策略、冲突解决 |
| 配置差异 | 集群特有配置 | 配置中心、环境变量 |
| 服务发现 | 跨集群调用 | 网关路由、DNS 策略 |
| 流量切换 | 灰度、蓝绿、金丝雀 | 路由规则、流量控制 |

### 9. 老 APP 兼容性
| 场景 | 问题 | 解决方案 |
|------|------|----------|
| API 降级 | 新接口老版本不支持 | 兼容层、Feature Flag |
| 功能开关 | 新功能老版本无 | 降级体验、静默失败 |
| 数据格式 | 新字段老版本忽略 | Optional 字段、默认值 |
| 安全策略 | 新协议老版本不支持 | 多协议支持、渐进升级 |

## 2. API 兼容性设计

### 2.1 版本策略
| 策略 | 说明 | 适用场景 |
|------|------|----------|
| URL 版本 | /v1/, /v2/ | 大版本变更 |
| Header 版本 | Accept: v2 | 小版本变更 |
| 参数版本 | ?version=2 | 可选升级 |

### 2.2 接口兼容清单
| 接口 | 变更类型 | 兼容方式 | 废弃计划 |
|------|----------|----------|----------|
| [接口1] | 字段新增 | Optional | - |
| [接口2] | 字段删除 | 保留字段+废弃标记 | 3个月后 |
| [接口3] | 逻辑变更 | Feature Flag | 灰度后 |

### 2.3 降级策略
| 场景 | 检测方式 | 降级行为 |
|------|----------|----------|
| 新接口失败 | 异常捕获 | 回退旧接口 |
| 字段缺失 | 空值判断 | 使用默认值 |
| 功能不可用 | 特性检测 | 静默失败/提示 |

## 3. 数据兼容性设计

### 3.1 数据库
#### 版本兼容
| 数据库 | 最低版本 | 特性差异 | 兼容处理 |
|--------|----------|----------|----------|
| MySQL | 5.7 | JSON字段、窗口函数 | 条件使用 |

#### Schema 变更
| 变更类型 | 策略 | 回滚方案 |
|----------|------|----------|
| 新增字段 | DEFAULT + Nullable | 删除字段 |
| 删除字段 | 先废弃 → 再删除 | 恢复字段 |
| 类型变更 | 扩大类型 (INT→BIGINT) | 缩小类型 |
| 索引变更 | 在线 DDL | 删除索引 |

### 3.2 缓存
| 场景 | Key 设计 | 迁移策略 | 过期策略 |
|------|----------|----------|----------|
| 结构变更 | user:{version}:{id} | 双写 → 切换 → 清理 | TTL + 主动清理 |
| 序列化变更 | 后缀区分 | 渐进刷新 | 版本过期 |

### 3.3 消息队列
| 场景 | 消息设计 | 消费策略 |
|------|----------|----------|
| 格式变更 | 版本号字段 | 多版本消费者 |
| Topic 变更 | 新旧 Topic 共存 | 双写 → 切换 |

## 4. 多集群兼容性设计

### 4.1 集群架构
```
┌─────────────────┐     ┌─────────────────┐
│    集群 A (新)    │     │    集群 B (旧)    │
│  ┌───────────┐  │     │  ┌───────────┐  │
│  │ Service X │  │     │  │ Service X │  │
│  └───────────┘  │     │  └───────────┘  │
│        │          │     │        │          │
│        ▼          │     │        ▼          │
│  ┌───────────┐  │     │  ┌───────────┐  │
│  │  MySQL    │  │     │  │  MySQL    │  │
│  └───────────┘  │     │  └───────────┘  │
└─────────────────┘     └─────────────────┘
        │                         │
        └───────── 数据同步 ────────┘
```

### 4.2 数据同步策略
| 策略 | 场景 | 实现 |
|------|------|------|
| 主从复制 | 读写分离 | MySQL 主从 |
| 双向同步 | 双活集群 | Canal + 消息队列 |
| 最终一致 | 跨集群缓存 | 消息队列 + 本地刷新 |

### 4.3 流量切换
| 方式 | 策略 | 回滚 |
|------|------|------|
| 蓝绿部署 | 整体切换 | DNS 切换 |
| 金丝雀 | 比例切换 | 调整比例 |
| 灰度发布 | 用户分批 | Feature Flag |

## 5. 老 APP 兼容性设计

### 5.1 版本支持范围
| APP 版本 | 用户占比 | 支持策略 | 废弃计划 |
|----------|----------|----------|----------|
| 最新版 | 70% | 全功能 | - |
| 上一个版本 | 20% | 核心功能 | 6个月后 |
| 更老版本 | 10% | 基础功能 | 3个月后 |

### 5.2 功能降级矩阵
| 功能 | 最新版 | 上一版本 | 更老版本 |
|------|--------|----------|----------|
| [功能1] | ✅ 完整 | ✅ 简化 | ❌ 不支持 |
| [功能2] | ✅ 完整 | ✅ 完整 | ⚠️ 降级 |

### 5.3 API 降级实现
```java
// 服务端兼容层
public Response handle(Request request) {
    String appVersion = request.getHeader("X-App-Version");
    
    if (isOldVersion(appVersion)) {
        return legacyHandler.handle(request);  // 旧版逻辑
    }
    return newHandler.handle(request);  // 新版逻辑
}
```

## 代码扫描规则

### API 版本识别
```java
// 识别版本化 API
@RequestMapping("/v1/users")  // 旧版本
@RequestMapping("/v2/users")  // 新版本

// 识别废弃标记
@Deprecated
@DeprecatedApi(since = "2.0")
```

### 数据库版本识别
```xml
<!-- pom.xml -->
<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-java</artifactId>
    <version>8.0.33</version>
</dependency>
```

### 缓存版本识别
```yaml
# application.yml
spring:
  redis:
    version: 6.x  # 识别 Redis 版本
```

### 多集群识别
```yaml
# application-prod.yml
spring:
  profiles:
    active: cluster-a  # 识别多集群配置
```
