| 等级 | 说明 | 适用数据 |

## 2. 认证设计

### 2.1 认证方式
| 方式 | 使用场景 | 说明 |
|------|----------|------|
| 账号密码 | Web登录 | 传统登录 |
| 手机验证码 | 移动端 | 快速登录 |
| OAuth2.0 | 第三方登录 | 微信/支付宝 |
| SSO | 企业用户 | 统一身份 |

### 2.2 Token设计
| 项目 | 内容 |
|------|------|
| **Token类型** | JWT / Session |
| **有效期** | 访问Token: 2h, 刷新Token: 7d |
| **存储位置** | localStorage / HttpOnly Cookie |
| **传输方式** | Header: Authorization: Bearer {token} |

### 2.3 JWT配置
| Claim | 值 | 说明 |
|-------|------|------|
| iss | [签发者] | 签发方 |
| sub | [用户ID] | 用户标识 |
| aud | [接收者] | 接收方 |
| exp | [过期时间] | 过期时间戳 |
| iat | [签发时间] | 签发时间戳 |

## 4. 数据安全

### 4.1 数据分类
| 数据类型 | 敏感级别 | 保护措施 |
|----------|----------|----------|
| 用户密码 | 绝密 | 加密存储, 不可逆 |
| 手机号 | 机密 | 加密存储, 脱敏显示 |
| 姓名 | 内部 | 脱敏显示 |
| 公开信息 | 公开 | 无 |

### 4.2 加密策略
| 场景 | 加密方式 | 说明 |
|------|----------|------|
| 密码存储 | BCrypt | 单向加密 |
| 敏感数据 | AES-256 | 可逆加密 |
| 传输加密 | TLS 1.3 | 传输层 |
| 数据库加密 | TDE | 存储加密 |

### 4.3 脱敏规则
| 数据类型 | 规则 | 示例 |
|----------|------|------|
| 手机号 | 中间4位* | 138****1234 |
| 身份证 | 保留前3后4 | 110***********1234 |
| 银行卡 | 保留后4位 | ************1234 |
| 姓名 | 保留姓 | 张** |

## 5. 接口安全

### 5.1 接口认证
| 接口类型 | 认证方式 | 说明 |
|----------|----------|------|
| 公开接口 | 无 | 无需认证 |
| 用户接口 | Bearer Token | JWT认证 |
| 管理接口 | API Key | 服务间调用 |
| 内部接口 | IP白名单 | 内网访问 |

### 5.2 防护措施
| 攻击类型 | 防护措施 | 说明 |
|----------|----------|------|
| SQL注入 | 参数化查询 | ORM/预编译 |
| XSS | 输入过滤 | 转义输出 |
| CSRF | Token校验 | 双重提交 |
| 重放攻击 | 时间戳+签名 | 请求签名 |
| 暴力破解 | 限流+验证码 | 登录保护 |

### 5.3 请求签名
```
sign = MD5(appKey + timestamp + nonce + appSecret)
```

| 参数 | 说明 |
|------|------|
| appKey | 应用标识 |
| timestamp | 时间戳, 5分钟有效 |
| nonce | 随机数, 防重放 |
| appSecret | 应用密钥 |
