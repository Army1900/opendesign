## 2. REST API设计

### 2.1 URL规范
```
/{version}/{resource}/{id}/{sub-resource}
```

| 规则 | 说明 | 示例 |
|------|------|------|
| 版本 | v开头 | /v1/ |
| 资源 | 复数形式 | /users/ |
| ID | 路径参数 | /users/123/ |
| 子资源 | 嵌套 | /users/123/orders/ |

### 2.2 HTTP方法规范
| 方法 | 用途 | 幂等性 | 示例 |
|------|------|--------|------|
| GET | 查询资源 | 是 | GET /users |
| POST | 创建资源 | 否 | POST /users |
| PUT | 全量更新 | 是 | PUT /users/123 |
| PATCH | 部分更新 | 否 | PATCH /users/123 |
| DELETE | 删除资源 | 是 | DELETE /users/123 |

### 2.3 请求规范
#### Query参数
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | integer | 否 | 页码, 默认1 |
| size | integer | 否 | 每页数量, 默认20 |
| sort | string | 否 | 排序字段 |
| order | string | 否 | asc/desc |

#### 请求头
| Header | 必填 | 说明 |
|--------|------|------|
| Authorization | 是 | Bearer {token} |
| Content-Type | 是 | application/json |
| X-Request-Id | 否 | 请求追踪ID |

### 2.4 响应规范
#### 成功响应
```json
{
  "code": 0,
  "data": {},
  "message": "success"
}
```

#### 分页响应
```json
{
  "code": 0,
  "data": {
    "items": [],
    "total": 100,
    "page": 1,
    "size": 20
  }
}
```

#### 错误响应
```json
{
  "code": 400001,
  "data": null,
  "message": "参数错误"
}
```

## 6. 错误处理

### 6.1 错误码规范
| 错误码范围 | 含义 | 示例 |
|------------|------|------|
| 0 | 成功 | 0 |
| 1xxxxx | 系统错误 | 100001 |
| 2xxxxx | 业务错误 | 200001 |
| 3xxxxx | 第三方错误 | 300001 |
| 4xxxxx | 客户端错误 | 400001 |

### 6.2 常见错误码
| 错误码 | 含义 | 说明 |
|--------|------|------|
| 400001 | 参数错误 | 请求参数校验失败 |
| 400002 | 格式错误 | 数据格式不正确 |
| 401001 | 未登录 | Token无效或过期 |
| 403001 | 无权限 | 权限不足 |
| 404001 | 不存在 | 资源不存在 |
| 500001 | 服务错误 | 内部服务异常 |

|------|------|------|

| 恢复 | 30s后半开 | 尝试恢复 |
