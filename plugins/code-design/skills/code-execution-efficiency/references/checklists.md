# 性能检查项清单

全链路视角的性能检查项，每个检查项都标注关联影响，避免单点分析。

---

## 代码层检查项

### 调用模式

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 循环内数据库调用 | for/while 中执行 SQL | N+1 问题、连接池耗尽、响应时间线性增长 | 🔴 高 |
| 循环内外部调用 | 循环中调用 RPC/HTTP | 串行累积延迟、超时风险 | 🔴 高 |
| 循环内缓存调用 | 循环中访问缓存 | 多次网络 IO、可批量优化 | 🟡 中 |
| 递归调用过深 | 深度递归无终止条件 | 栈溢出、性能下降 | 🔴 高 |
| 反射频繁使用 | 循环中使用反射 | 性能损耗、CPU 高 | 🟡 中 |

### 并发与锁

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 同步阻塞可并行操作 | 多个独立操作串行执行 | 响应时间累加 | 🟡 中 |
| 锁粒度过粗 | 大范围 synchronized/lock | 并发吞吐下降、线程等待 | 🔴 高 |
| 死锁风险 | 多锁顺序不一致 | 线程阻塞、服务不可用 | 🔴 高 |
| 线程池配置不当 | 核心线程数/队列/拒绝策略 | 任务堆积或资源浪费 | 🟡 中 |
| 线程泄漏 | 线程创建未回收 | 资源耗尽、OOM | 🔴 高 |

### 内存与对象

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 大对象频繁创建 | 循环中 new 大对象 | GC 频繁、内存碎片 | 🟡 中 |
| 对象池未使用 | 可复用对象每次新建 | 内存分配开销 | 🟢 低 |
| 集合初始容量未设 | ArrayList/HashMap 扩容 | 频繁扩容、内存浪费 | 🟢 低 |
| 流未关闭 | IO 流未 try-with-resources | 资源泄漏 | 🔴 高 |
| 内存泄漏 | 静态集合持续增长 | OOM、Full GC | 🔴 高 |

### 异常处理

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 异常用于流程控制 | 用异常控制业务逻辑 | 性能损耗、栈追踪开销 | 🟡 中 |
| 捕获后静默 | catch 空块或仅打印 | 问题隐藏、排查困难 | 🟡 中 |
| 重试无限制 | 异常后无限重试 | 级联失败、资源耗尽 | 🔴 高 |

## 数据层检查项

### SQL 执行计划

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 全表扫描 | EXPLAIN type=ALL | 大数据量下 CPU/IO 飙升 | 🔴 高 |
| 索引失效 | 函数/隐式转换/前缀模糊 | 同上 | 🔴 高 |
| 索引选择错误 | 优化器选错索引 | 慢查询 | 🟡 中 |
| 回表过多 | Using index condition + 大量回表 | 随机 IO、响应慢 | 🟡 中 |
| 临时表 | Using temporary | 内存/磁盘临时表开销 | 🟡 中 |
| 文件排序 | Using filesort + 大数据量 | 排序开销大 | 🟡 中 |

### SQL 写法

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| SELECT * | 查询不需要的字段 | 网络传输、内存占用、回表 | 🟡 中 |
| 大 OFFSET | LIMIT 10000, 10 | 扫描大量无关行 | 🔴 高 |
| OR 条件 | 多字段 OR | 索引可能失效 | 🟡 中 |
| 子查询 | 嵌套子查询 | 执行计划复杂、可能全表 | 🟡 中 |
| DISTINCT | 大数据量去重 | 排序+临时表 | 🟡 中 |
| LIKE 前缀模糊 | LIKE '%xxx' | 索引失效 | 🔴 高 |

### 事务与锁

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 长事务 | 多 SQL 在同一事务 | 锁持有时间长、连接占用 | 🔴 高 |
| 锁等待 | FOR UPDATE 长时间持有 | 并发吞吐下降、超时 | 🔴 高 |
| 死锁 | 多表锁顺序不一致 | 事务失败、需要重试 | 🔴 高 |
| 事务范围过大 | 非必要操作在事务内 | 连接占用时间长 | 🟡 中 |
| 无事务控制 | 应该事务的操作没有 | 数据不一致 | 🔴 高 |

### 连接与资源

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 连接池过小 | 最大连接数不足 | 请求排队、超时 | 🔴 高 |
| 连接池过大 | 最大连接数过多 | DB 连接数耗尽、资源浪费 | 🟡 中 |
| 连接泄漏 | 获取连接未归还 | 连接池耗尽 | 🔴 高 |
| 超时未配置 | 查询无超时限制 | 慢查询拖垮服务 | 🔴 高 |

### 批量操作

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 单条循环插入 | for 循环单条 INSERT | 大量网络 IO、事务开销 | 🔴 高 |
| 大批量单次提交 | 10 万条一次提交 | 长事务、内存溢出 | 🔴 高 |
| 批量大小不当 | batch size 过大或过小 | 性能次优 | 🟡 中 |

## 缓存层检查项

### 缓存使用

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 缓存穿透 | 查不存在的 key，无空值缓存 | 绕过缓存压 DB | 🔴 高 |
| 缓存击穿 | 热点 key 过期，并发重建 | 瞬间 DB 压力 | 🔴 高 |
| 缓存雪崩 | 大量 key 同时过期 | DB 崩溃 | 🔴 高 |
| 缓存命中率低 | 缓存基本无效 | 缓存形同虚设 | 🟡 中 |
| 只读不写 | 读缓存但更新时未失效 | 数据不一致 | 🔴 高 |

### Redis 特定

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 大 Key | value > 10KB | 序列化慢、网络 IO、阻塞 | 🔴 高 |
| 热点 Key | 单 key 高频访问 | 单节点瓶颈 | 🟡 中 |
| 慢命令 | KEYS/Sort/HGETALL 大 hash | 阻塞单线程 | 🔴 高 |
| 连接池不足 | maxTotal 过小 | 请求排队 | 🟡 中 |
| 未设过期 | key 无 TTL | 内存泄漏 | 🟡 中 |
| 序列化低效 | Java 序列化/大 JSON | CPU 和网络开销 | 🟡 中 |

## 外部服务检查项

### 超时与重试

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 无超时配置 | 调用无 timeout | 级联超时、线程阻塞 | 🔴 高 |
| 超时配置过长 | timeout > 业务 SLA | 响应慢、用户体验差 | 🟡 中 |
| 超时配置过短 | timeout < 正常响应 | 误杀正常请求 | 🟡 中 |
| 重试无限制 | 失败后无限重试 | 级联放大、雪崩 | 🔴 高 |
| 重试间隔不合理 | 重试过快或过慢 | 加剧压力或延迟过长 | 🟡 中 |

### 熔断与降级

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 无熔断机制 | 下游故障持续调用 | 资源耗尽、级联失败 | 🔴 高 |
| 无降级策略 | 依赖服务不可用时无备选 | 功能完全不可用 | 🟡 中 |
| 无限流保护 | 无请求量限制 | 被打垮、雪崩 | 🔴 高 |

### 连接管理

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 连接不复用 | 每次 new HTTP 连接 | 连接建立开销大 | 🟡 中 |
| 连接池过小 | 最大连接数不足 | 请求排队、超时 | 🔴 高 |
| Keep-Alive 未启用 | 每次新建 TCP | 连接开销 | 🟡 中 |
| DNS 缓存未配置 | 频繁 DNS 解析 | 延迟增加 | 🟢 低 |

### 调用模式

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 同步调用可异步 | 可异步的同步调用 | 响应时间累加 | 🟡 中 |
| 串行调用可并行 | 多个独立调用串行 | 响应时间累加 | 🟡 中 |
| 无请求去重 | 相同请求重复发送 | 资源浪费 | 🟢 低 |

## 并发与高负载检查项

### 线程模型

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 线程数过多 | 线程池过大 | 上下文切换开销、CPU 争用 | 🟡 中 |
| 线程数过少 | 线程池过小 | 吞吐量受限 | 🟡 中 |
| 队列过长 | 任务队列过大 | 响应延迟增加、内存占用 | 🟡 中 |
| 拒绝策略不当 | 拒绝策略不合适 | 任务丢失或异常 | 🟡 中 |

### 资源竞争

| 检查项 | 说明 | 关联影响 | 风险 |
|--------|------|----------|------|
| 热点数据竞争 | 同一数据高并发更新 | 锁等待、吞吐下降 | 🔴 高 |

| 限流不均匀 | 限流配置不当 | 部分节点过载 | 🟡 中 |

### 背压与流控

|--------|------|----------|------|
| 无背压机制 | 生产快于消费 | 内存溢出、系统崩溃 | 🔴 高 |
| 无速率限制 | 无 QPS 限制 | 被突发流量打垮 | 🔴 高 |
| 队列无界 | 队列无上限 | 内存溢出 | 🔴 高 |

## 使用方法

### 1. 选择合适的检查清单

根据分析场景选择：
- **慢查询**：重点看「数据层」+「代码层调用模式」
- **接口慢**：全链路检查，从「代码层」到「外部服务」
- **高并发问题**：重点看「并发与高负载」+「数据层事务与锁」

### 2. 关联分析

每个检查项都要看关联影响：
```
发现：循环内数据库调用
  ↓ 关联
  - 数据层：每次 SQL 是否走索引？
  - 缓存层：是否可以先查缓存？
  - 连接池：连接池是否足够？
```

### 3. 输出格式

```markdown
## 检查项结果

| 检查项 | 位置 | 状态 | 风险 | 说明 |
|--------|------|------|------|------|
| 循环内数据库调用 | OrderService:67 | ❌ | 🔴 | 循环查询订单详情 |
| 索引失效 | order_detail.xml:23 | ❌ | 🔴 | DATE 函数导致索引失效 |

**风险统计**: 🔴 高风险 2 项 | 🟡 中风险 0 项 | 🟢 低风险 0 项
```
