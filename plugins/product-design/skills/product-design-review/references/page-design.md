# 页面设计分析模块

扫描设计稿/原型中的所有页面，逐页描述元素布局和组成，输出结构化的页面设计文档。

## 分析目标

将设计稿中的每个页面「翻译」成结构化的元素描述，让不看过原稿的人也能清楚知道每个页面有哪些区域、每个区域有哪些元素。

> 核心原则：**只描述页面的元素布局和组成，不涉及业务逻辑、交互行为、数据流转**。业务逻辑归「业务逻辑」章节，交互行为归「交互设计」章节。

## 页面类型分类

| 页面类型 | 特征 | 必有区域 |
|----------|------|----------|
| 列表页 | 数据展示 + 搜索 + 分页 | 搜索区、表格区、分页 |
| 表单页 | 数据录入 + 提交 | 表单字段、操作按钮 |
| 详情页 | 数据展示 + 只读 | 信息区块、操作按钮 |
| 仪表盘 | 数据可视化 + 统计 | 卡片/图表区域 |
| 弹窗 | 辅助操作 + 快速交互 | 弹窗表单、确认/取消 |
| 抽屉 | 侧滑展示 | 抽屉内容、底部操作 |
| 标签页 | 多Tab内容切换 | Tab导航、内容区域 |
| 向导页 | 步骤引导 | 步骤条、表单区域 |

## 描述规范

### 元素类型对照表

| 控件类型 | 描述要点 |
|----------|----------|
| 输入框 | placeholder、最大长度、是否必填、校验规则 |
| 下拉选择 | 选项来源（字典/接口）、是否可搜索、是否多选 |
| 日期选择 | 格式、是否范围选择、默认值 |
| 数字输入 | 最小值、最大值、步长、单位 |
| 文本域 | 行数、最大长度、是否可拖拽 |
| 开关/复选框 | 默认值、选中/未选中含义 |
| 文件上传 | 文件类型限制、大小限制、数量限制 |
| 级联选择 | 层级来源、是否可多选 |
| 树选择 | 数据来源、是否可搜索 |

### 区域命名规范

使用描述性名称，不使用抽象编号：

| 好的命名 | 不好的命名 |
|----------|------------|
| 搜索区域 | 区域A |
| 表格上方操作栏 | 操作区 |
| 新增用户弹窗 | 弹窗1 |
| 基本信息区块 | 区块1 |
| 底部统计栏 | 底部区域 |

## 分析步骤

1. **扫描全部页面**：遍历设计稿中所有页面/状态
2. **识别页面类型**：判断每个页面属于哪种类型
3. **划分页面区域**：将每个页面拆分为功能区域
4. **描述区域元素**：逐个元素记录类型、属性、校验规则
5. **记录弹窗/抽屉**：单独描述每个弹窗/抽屉的内容
6. **汇总通用组件**：跨页面复用的组件归纳

## 注意事项

- **只描述页面结构**，不写业务规则（如"用户名不能重复"属于业务逻辑）
- **只描述元素组成**，不写交互行为（如"点击按钮后跳转"属于交互设计）
- **只描述页面内容**，不写数据来源（如"数据从接口获取"属于技术实现）
- **保留原始文案**：按钮文字、列名、字段名保持原文，不要概括
- **标注来源**：每个页面标注来自哪个设计稿文件
