# 页面设计分析模块

扫描设计稿/原型中的实际页面，将每个页面的元素布局如实记录为结构化描述。

## 核心原则

**只记录设计稿中实际存在的内容，禁止自行编造任何元素。**

> 这是最重要的原则。页面设计的价值在于忠实还原设计稿的每一个元素。如果设计稿中没有某个元素，就不要写。如果不确定是否存在，标注「[待确认]」而不是猜测。

**三大禁止**：
1. **禁止凭空编造**：不允许自己想象页面元素、搜索条件、表格列、弹窗字段
2. **禁止套用模板**：不允许用通用的列表页/弹窗模板填充，每个页面必须如实反映原稿
3. **禁止概括省略**：不允许把3个搜索条件概括为"若干搜索条件"

**动作判定原则**：原型/设计稿画的是页面当前状态，不代表所有页面都是全新做的。每个页面、每个组件的动作类型（新增/修改/优化）必须根据需求文档中的实际描述来确定：
- 需求明确说是新做的 → 标记为「新增」
- 需求说在已有页面上改 → 标记为「修改」
- 需求说优化已有功能 → 标记为「优化」
- 需求没有明确说 → 标注「[待确认]」

## 页面区域拆解规则（强制遵循）

**每个页面必须按功能区域拆解为编号子章节，禁止把所有元素混在一起写。**

按以下区域类型顺序逐个检查，设计稿中存在的区域才写子章节，不存在的跳过：

| 区域类型 | 包含内容 | 适用页面 |
|----------|----------|----------|
| 搜索区域 | 搜索条件表格 + 搜索按钮表格 | 列表页 |
| 表格区域 | 表格上方按钮表格 + 表格列表格 + 操作列按钮表格 + 分页 | 列表页 |
| 表单区域 | 表单字段表格 + 校验规则 + 提交按钮 | 表单页 |
| 信息展示区域 | 按区块分组的展示字段表格 | 详情页 |
| Tab区域 | Tab标签 + 各Tab下的内容 | 有Tab的页面 |
| 弹窗/抽屉 | 弹窗标题 + 弹窗按钮表格 + 弹窗字段表格 | 有弹窗的页面 |

**拆解流程（每个页面必须执行）**：
1. 确定页面类型（列表页/表单页/详情页/混合页）
2. 按上表顺序逐个检查设计稿中存在哪些区域
3. 每个存在的区域生成一个编号子章节（**1. 搜索区域**、**2. 表格区域**、**3. XX弹窗** ...）
4. 每个子章节内用表格列出具体元素（搜索条件名、列名、字段名、按钮文字等）

**列表页拆解示例**：
- **1. 搜索区域** → 表格列出每个搜索条件 + 表格列出搜索按钮
- **2. 表格区域** → 表格列出表格上方按钮 + 表格列出每列 + 表格列出操作列按钮 + 分页
- **3. 新增XX弹窗** → 弹窗标题 + 表格列出弹窗按钮 + 表格列出每个字段
- **4. 编辑XX弹窗** → 同上

**详情页拆解示例**：
- **1. 基本信息区域** → 表格列出每个展示字段（名称、展示方式、说明）
- **2. XX信息区域** → 按设计稿实际分区

**表单页拆解示例**：
- **1. 表单字段** → 表格列出每个字段（名称、控件类型、必填、校验规则、说明）
- **2. XX弹窗**（如有）→ 同弹窗格式

## 输出模板（强制格式，必须严格照此输出）

**以下格式是强制要求，不是参考示例。** 每个页面必须先拆解为功能区域子章节，每个子章节内用表格列出具体元素。不得跳过区域拆解直接写元素，不得用段落描述代替表格。

每个子章节标题 = **编号. 动作 + 区域/组件名**，动作根据需求文档确定。

### 子标题命名规则

子标题 = **编号. 动作 + 区域/组件名**，动作来自需求文档：

| 正确 | 错误 |
|------|------|
| `1. 新增搜索区域` | `搜索条件` |
| `2. 修改表格区域` | `用户列表表格` |
| `3. 新增用户弹窗` | `弹窗` |
| `4. 修改编辑用户弹窗` | `编辑弹窗` |
| `1. 新增基本信息区域` | `基本信息` |

## 分析步骤

1. **确定页面类型**：先判断这个页面是列表页、表单页、详情页还是混合页
2. **按区域拆解**：对照「页面区域拆解规则」中的区域类型，逐个检查设计稿中存在哪些区域
3. **逐个区域记录元素**：每个区域用表格列出具体元素（字段名、列名、按钮文字保持设计稿原文）
4. **保留原文**：字段名、按钮文字、列名保持设计稿中的原文
5. **标注来源**：每个页面标注来自哪个设计稿文件

## 常见错误（必须避免）

| 错误 | 正确做法 |
|------|----------|
| 不做区域拆解，所有元素混在一起 | 必须拆解为搜索区域、表格区域、弹窗等子章节 |
| 用段落描述代替表格（"搜索区有用户名、状态等条件"） | 必须用表格逐个列出每个元素 |
| 设计稿有2个搜索条件，自己编了5个 | 只写设计稿中的2个 |
| 设计稿没有弹窗，自己加了一个"新增弹窗" | 设计稿没有就不写 |
| 搜索条件写成"若干搜索条件" | 逐个列出每个搜索条件的名称和类型 |
| 弹窗字段写成"包含基本信息字段" | 逐个列出每个字段的名称、控件类型、是否必填 |
| 设计稿中没有标注必填，自己标记了必填 | 没有明确信息的标注「[待确认]」 |
| 按钮写成"有新增、导出等按钮" | 用表格逐个列出按钮名称、操作类型、说明 |
| 子标题不带动作词（"搜索条件"、"弹窗"） | 子标题必须带动作：新增搜索区域、修改XX弹窗 |
