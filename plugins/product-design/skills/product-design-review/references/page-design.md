# 页面设计分析模块

扫描设计稿/原型中的实际页面，将每个页面的元素布局如实记录为结构化描述。

## 核心原则

**只记录设计稿中实际存在的内容，禁止自行编造任何元素。**

> 这是最重要的原则。页面设计的价值在于忠实还原设计稿的每一个元素。如果设计稿中没有某个元素，就不要写。如果不确定是否存在，标注「[待确认]」而不是猜测。

**三大禁止**：
1. **禁止凭空编造**：不允许自己想象页面元素、搜索条件、表格列、弹窗字段
2. **禁止套用模板**：不允许用通用的列表页/弹窗模板填充，每个页面必须如实反映原稿
3. **禁止概括省略**：不允许把3个搜索条件概括为"若干搜索条件"

**动作判定原则**：原型/设计稿画的是页面当前状态，不代表所有页面都是全新做的。每个页面、每个组件的动作类型（新增/修改/优化）必须根据需求文档中的实际描述来确定：
- 需求明确说是新做的 → 标记为「新增」
- 需求说在已有页面上改 → 标记为「修改」
- 需求说优化已有功能 → 标记为「优化」
- 需求没有明确说 → 标注「[待确认]」

## 页面区域拆解规则（强制遵循）

**每个页面必须按功能区域拆解为 `####` 级子标题，每个子区域用 `#####` 级子标题，禁止把所有元素混在一起写，禁止用粗体代替标题。**

按以下区域类型顺序逐个检查，设计稿中存在的区域才写子标题，不存在的跳过：

| 区域类型 | 子区域 | 适用页面 |
|----------|--------|----------|
| 搜索区域 | 搜索条件、搜索按钮 | 列表页 |
| 表格区域 | 表格上方按钮、表格列、操作列、分页 | 列表页 |
| 表单区域 | 表单字段、提交按钮 | 表单页 |
| 信息展示区域 | 按区块分组的展示字段 | 详情页 |
| Tab区域 | Tab标签、各Tab下的内容 | 有Tab的页面 |
| 弹窗/抽屉 | 弹窗按钮、弹窗字段 | 有弹窗的页面 |

**拆解流程（每个页面必须执行）**：
1. 确定页面类型（列表页/表单页/详情页/混合页）
2. 按上表顺序逐个检查设计稿中存在哪些区域
3. 每个存在的区域生成一个 `####` 级子标题（`#### 1. 新增搜索区域`）
4. 每个子区域生成一个 `#####` 级子标题（`##### 1.1 新增搜索条件`）
5. 每个子区域内用列表逐个列出具体元素

**列表页拆解示例**：
- `#### 1. 新增搜索区域` → `##### 1.1 新增搜索条件` + `##### 1.2 新增搜索按钮`
- `#### 2. 新增表格区域` → `##### 2.1 新增表格上方按钮` + `##### 2.2 新增表格列` + `##### 2.3 新增操作列`
- `#### 3. 新增用户弹窗` → `##### 3.1 新增弹窗按钮` + `##### 3.2 新增弹窗字段`
- `#### 4. 修改编辑用户弹窗` → 同上

**详情页拆解示例**：
- `#### 1. 新增基本信息区域` → 列表列出每个展示字段
- `#### 2. 新增XX区域` → 按设计稿实际分区

**表单页拆解示例**：
- `#### 1. 新增表单字段` → 列表列出每个字段
- `#### 2. 新增XX弹窗`（如有）→ 同弹窗格式

## 输出模板（强制格式，必须严格照此输出）

**以下格式是强制要求，不是参考示例。** 使用 `####`/`#####` 标题层级按章节输出，不得用粗体代替标题，不得用表格代替章节。每个元素用列表项逐个列出。

标题命名 = `####`/`#####` + `编号. 动作 + 区域/组件名`，动作根据需求文档确定。

### 标题层级规则

页面设计章节使用以下标题层级，**不得用粗体代替标题**：

| 层级 | 格式 | 用途 | 示例 |
|------|------|------|------|
| `###` | `5.X 动作+页面名` | 页面级 | `### 5.2 新增用户列表页` |
| `####` | `序号. 动作+区域名` | 功能区域级 | `#### 1. 新增搜索区域` |
| `#####` | `序号.序号 动作+子区域名` | 子区域级 | `##### 1.1 新增搜索条件` |

**标题命名 = 动作 + 区域/组件名**，动作来自需求文档：

| 正确 | 错误 |
|------|------|
| `#### 1. 新增搜索区域` | `**1. 搜索区域**`（禁止用粗体） |
| `##### 1.1 新增搜索条件` | `搜索条件`（缺少编号和动作） |
| `#### 2. 新增表格区域` | `#### 2. 表格`（缺少动作词） |
| `#### 3. 新增用户弹窗` | `**3. 弹窗**`（禁止用粗体） |
| `##### 3.2 新增弹窗字段` | `弹窗字段`（缺少编号和动作） |
| `#### 1. 新增基本信息区域` | `基本信息`（缺少编号和动作） |

## 分析步骤

1. **确定页面类型**：先判断这个页面是列表页、表单页、详情页还是混合页
2. **按区域拆解**：对照「页面区域拆解规则」中的区域类型，逐个检查设计稿中存在哪些区域
3. **生成标题层级**：区域用 `####`，子区域用 `#####`
4. **逐个列出元素**：每个子区域内用列表项（`- **元素名**：属性`）逐个列出
5. **保留原文**：字段名、按钮文字、列名保持设计稿中的原文
6. **标注来源**：每个页面标注来自哪个设计稿文件

## 常见错误（必须避免）

| 错误 | 正确做法 |
|------|----------|
| 用粗体代替标题（`**1. 搜索区域**`） | 必须用 `#### 1. 新增搜索区域` |
| 用表格列出所有元素 | 必须按章节输出，用列表逐个列出 |
| 不做区域拆解，所有元素混在一起 | 必须拆解为搜索区域、表格区域、弹窗等章节 |
| 用段落概括（"搜索区有用户名、状态等条件"） | 必须逐个列出每个元素 |
| 设计稿有2个搜索条件，自己编了5个 | 只写设计稿中的2个 |
| 设计稿没有弹窗，自己加了一个"新增弹窗" | 设计稿没有就不写 |
| 搜索条件写成"若干搜索条件" | 逐个列出每个搜索条件的名称和类型 |
| 弹窗字段写成"包含基本信息字段" | 逐个列出每个字段的名称、控件类型、是否必填 |
| 标题不带动作词（"搜索条件"、"弹窗"） | 标题必须带动作：新增搜索条件、修改XX弹窗 |
| 标题不带编号（"搜索区域"） | 标题必须带编号：`#### 1. 新增搜索区域` |
