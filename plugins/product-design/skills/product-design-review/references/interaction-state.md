# 状态设计分析模块

分析产品的状态设计，检查各种状态覆盖是否完整。

**每个章节必须包含两部分**：
1. **当前设计梳理**：从设计稿中提取的现有状态设计
2. **审查意见**：基于梳理内容给出的问题、风险、建议

## 检查项清单

| 检查项 | 说明 | 风险等级 | 常见问题 |
|--------|------|----------|----------|
| 默认态 | 正常显示状态 | 低 | - |
| 悬停态 | 鼠标悬停效果 | 低 | 无悬停效果 |
| 点击态 | 点击/按下效果 | 高 | 无点击反馈 |
| 选中态 | 选中/激活效果 | 中 | 无法区分选中 |
| 禁用态 | 不可用效果 | 高 | 禁用状态不清 |
| 加载态 | 加载中效果 | 高 | 无加载提示 |
| 错误态 | 验证失败效果 | 高 | 错误无提示 |
| 空状态 | 无数据效果 | 中 | 空白页面 |
| 一致性 | 同类元素状态一致 | 中 | 状态不统一 |
