# 检查项清单

各维度的详细检查项，用于逐项对照分析。

---

## 1. 高可用性检查

### 1.1 单点故障

| 检查项 | 说明 | 风险等级 | 常见问题 |
|--------|------|----------|----------|
| 核心服务多实例 | 关键服务至少 2 实例 | 🔴 P0 | 单实例部署 |
| 数据库主从 | 读高可用 + 故障切换 | 🔴 P0 | 单库无备份 |
| 缓存高可用 | Redis 主从/哨兵/集群 | 🔴 P0 | 单机 Redis |
| 消息队列高可用 | 多节点集群 | 🟠 P1 | 单机 MQ |
| 负载均衡 | 多后端 + 健康检查 | 🟠 P1 | 单 Nginx |

### 1.2 故障转移

| 检查项 | 说明 | 风险等级 | 常见问题 |
|--------|------|----------|----------|
| 数据库自动切换 | MHA/Orchestrator 等 | 🔴 P0 | 手动切换 |
| 缓存故障转移 | 哨兵/集群自动切换 | 🟠 P1 | 手动介入 |
| 服务熔断降级 | Hystrix/Sentinel | 🟠 P1 | 无熔断保护 |
| 限流保护 | 防止雪崩 | 🟠 P1 | 无限流 |

### 1.3 容灾能力

| 检查项 | 说明 | 风险等级 | 常见问题 |
|--------|------|----------|----------|
| 跨可用区部署 | 多 AZ 部署 | 🟠 P1 | 单机房 |
| 异地容灾 | 跨地域灾备 | 🟡 P2 | 无异地 |
| RTO/RPO | 恢复时间和数据丢失 | 🔴 P0 | 未定义 |
| 容灾演练 | 定期验证 | 🟡 P2 | 从未演练 |

## 2. 性能配置检查

### 2.1 连接池配置

| 检查项 | 说明 | 风险等级 | 合理范围 |
|--------|------|----------|----------|
| 数据库连接池大小 | max_pool_size | 🟠 P1 | (核心数*2) + 有效磁盘数 |
| 连接超时时间 | connection_timeout | 🟠 P1 | 3-5s |
| 空闲连接回收 | idle_timeout | 🟡 P2 | 10-30min |
| 最大生命周期 | max_lifetime | 🟡 P2 | 30min |
| Redis 连接池 | maxTotal | 🟠 P1 | 根据QPS调整 |

### 2.2 超时配置

| 检查项 | 说明 | 风险等级 | 常见问题 |
|--------|------|----------|----------|
| HTTP 连接超时 | connect_timeout | 🟠 P1 | 过长或无超时 |
| HTTP 读超时 | read_timeout | 🟠 P1 | 无超时导致阻塞 |
| 数据库查询超时 | statement_timeout | 🟠 P1 | 无超时慢查询 |
| RPC 超时 | dubbo/feign timeout | 🟠 P1 | 级联超时 |

### 2.3 线程池配置

| 检查项 | 说明 | 风险等级 | 常见问题 |
|--------|------|----------|----------|
| 核心线程数 | core_pool_size | 🟡 P2 | 过小或过大 |
| 最大线程数 | max_pool_size | 🟡 P2 | 配置不合理 |
| 队列容量 | queue_capacity | 🟠 P1 | 无界队列 OOM |
| 拒绝策略 | rejected_policy | 🟠 P1 | 无策略丢弃任务 |

### 2.4 JVM 配置

| 检查项 | 说明 | 风险等级 | 常见问题 |
|--------|------|----------|----------|
| 堆内存大小 | -Xms/-Xmx | 🟠 P1 | 设置相同，避免动态调整 |
| GC 策略 | G1/CMS/ZGC | 🟡 P2 | 选择不当 |
| GC 停顿目标 | MaxGCPauseMillis | 🟡 P2 | 未设置 |
| 元空间大小 | MetaspaceSize | 🟡 P2 | 默认值过小 |

## 3. 数据库检查

### 3.1 数据库配置

| 检查项 | 说明 | 风险等级 | 常见问题 |
|--------|------|----------|----------|
| max_connections | 最大连接数 | 🟠 P1 | 过小导致连接等待 |
| innodb_buffer_pool | 缓冲池大小 | 🟠 P1 | 过小影响性能 |
| innodb_log_file_size | 日志文件大小 | 🟡 P2 | 过小频繁刷盘 |
| slow_query_log | 慢查询日志 | 🟡 P2 | 未开启 |
| long_query_time | 慢查询阈值 | 🟡 P2 | 阈值过大 |

### 3.2 数据量与增长

| 检查项 | 说明 | 风险等级 | 风险阈值 |
|--------|------|----------|----------|
| 单表数据量 | 过大影响查询 | 🟠 P1 | > 1000 万行 |
| 日增量 | 数据增长速度 | 🟡 P2 | > 100 万/天 |
| 磁盘使用率 | 空间预警 | 🔴 P0 | > 80% |
| 索引数量 | 索引过多 | 🟡 P2 | > 5 个/表 |

### 3.3 主从配置

| 检查项 | 说明 | 风险等级 | 常见问题 |
|--------|------|----------|----------|
| 主从复制状态 | 是否正常 | 🔴 P0 | 复制中断 |
| 主从延迟 | 秒级延迟 | 🟠 P1 | > 1s |
| 只读模式 | 从库只读 | 🟡 P2 | 从库可写 |
| GTID 模式 | 增强复制可靠性 | 🟡 P2 | 未开启 |

## 风险等级定义

| 等级 | 定义 | 处理时限 | 示例 |
|------|------|----------|------|
| 🔴 P0 | 可能导致服务不可用或数据丢失 | 立即处理 | 单点故障、无备份 |
| 🟠 P1 | 影响性能或存在隐患 | 1-2 周内 | 配置不合理、缺少监控 |
| 🟡 P2 | 存在改进空间 | 迭代排期 | 规范性、最佳实践 |
| 🟢 P3 | 优化建议 | 可选 | 架构演进建议 |
