#### 3. REST API设计

##### URL规范
```
/{version}/{resource}/{id}/{sub-resource}
```
| 规则 | 说明 | 示例 |
|------|------|------|
| 版本 | v开头 | /v1/ |
| 资源 | 复数形式 | /users/ |
| ID | 路径参数 | /users/123/ |

##### HTTP方法规范
| 方法 | 用途 | 幂等性 | 示例 |
|------|------|--------|------|
| GET | 查询资源 | 是 | GET /users |
| POST | 创建资源 | 否 | POST /users |
| PUT | 全量更新 | 是 | PUT /users/123 |
| PATCH | 部分更新 | 否 | PATCH /users/123 |
| DELETE | 删除资源 | 是 | DELETE /users/123 |

##### 请求规范
| Header | 必填 | 说明 |
|--------|------|------|
| Authorization | 是 | Bearer {token} |
| Content-Type | 是 | application/json |
| X-Request-Id | 否 | 请求追踪ID |

##### Query参数
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | integer | 否 | 页码, 默认1 |
| size | integer | 否 | 每页数量, 默认20 |
| sort | string | 否 | 排序字段 |

##### 响应规范
```json
// 成功响应
{
  "code": 0,
  "data": {},
  "message": "success"
}

// 分页响应
{
  "code": 0,
  "data": {
    "items": [],
    "total": 100,
    "page": 1,
    "size": 20
  }
}

// 错误响应
{
  "code": 400001,
  "data": null,
  "message": "参数错误"
}
```

## 检查项清单

### API 设计检查项

| 检查项 | 说明 | 风险等级 | 常见问题 |
|--------|------|----------|----------|
| RESTful 规范 | URL/HTTP 方法语义正确 | 低 | 动词化 URL |
| 版本管理 | URL/Header 版本控制 | 中 | 无版本导致兼容问题 |
| 统一响应格式 | code/data/message 结构 | 低 | 响应格式不统一 |
| 错误码规范 | 分层错误码定义 | 中 | 错误码混乱 |
| 分页规范 | page/size/cursor 分页 | 中 | 无分页或分页不规范 |
| 排序过滤 | 统一排序和过滤参数 | 低 | 参数不统一 |
| 认证授权 | Token/API Key 校验 | 高 | 无认证或认证不完整 |
| 权限控制 | 接口级权限校验 | 高 | 越权访问 |
| 限流保护 | 全局/用户/接口限流 | 高 | 无限流保护 |
| 熔断降级 | 外部依赖熔断 | 高 | 无熔断保护 |
| 请求追踪 | RequestId 链路追踪 | 中 | 无追踪排查困难 |
| 幂等设计 | POST 接口幂等性 | 高 | 重复提交 |
| 参数校验 | 入参格式/范围校验 | 中 | 无校验 |
| 响应压缩 | Gzip/Brotli 压缩 | 低 | 大响应无压缩 |

### 事件设计检查项

| 检查项 | 说明 | 风险等级 | 常见问题 |
|--------|------|----------|----------|
| 事件分类 | 领域事件/集成事件分离 | 中 | 事件边界不清 |
| 事件结构 | eventId/type/data/version | 中 | 结构不统一 |
| Topic 命名 | 统一命名规范 | 低 | Topic 命名混乱 |
| 消费者组 | 正确配置消费者组 | 高 | 重复消费 |
| 消费语义 | 至少一次/精确一次 | 高 | 消息丢失 |
| 幂等消费 | eventId 去重 | 高 | 重复消费问题 |
| 顺序保证 | 分区 Key 设计 | 高 | 乱序问题 |
| 死信处理 | 失败消息死信队列 | 高 | 消息丢失 |
| 重试策略 | 重试次数和间隔 | 高 | 无限重试 |
| 生产者确认 | ACK 确认机制 | 高 | 发送无确认 |
| 消费者确认 | 手动提交 Offset | 高 | 自动提交丢失 |
| 消息持久化 | 持久化到磁盘 | 高 | 重启丢失 |
| 消息积压 | 积压监控和告警 | 中 | 积压无感知 |
| 事件版本 | 新老版本兼容 | 高 | 格式不兼容 |
