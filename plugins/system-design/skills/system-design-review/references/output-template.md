# 输出文档结构模板

报告的完整文档结构，包含每个章节的具体格式。

## 核心原则

每个章节必须包含「当前设计梳理」+「审查意见」两部分。

## 完整文档结构

```markdown
# [项目名] 系统设计审查报告

> 文档生成时间：YYYY-MM-DD HH:mm
> 数据来源：[设计文档路径] + [代码库路径]
> 审查范围：[完整审查/选择性审查：xxx]

不要输出章节目录，直接从正文开始。
```

## 状态和风险定义

**状态标记**:
- ✅ 已设计：设计文档和代码实现一致
- ⚠️ 部分设计：有设计但未实现，或实现与设计不一致
- ❌ 缺失：无设计文档且无代码实现

**风险等级**:
- 🔴 高风险：可能导致系统故障或安全问题
- 🟡 中风险：影响系统可维护性或扩展性
- 🟢 低风险：仅影响规范性和一致性

## 文档格式要求

- 每个 chapter 使用 `##` 标题
- 使用表格展示检查项
- 每个模块输出风险统计
- 最终汇总所有风险项
