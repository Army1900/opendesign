# 使用场景分析 Skill

基于用户提供的文档，分析并输出标准化的使用场景文档。

## 输出路径

```
docs/product-design/[项目名]/使用场景.md
```

## 工作流程

### 1. 接收输入
- HTML 文件夹 / 压缩包 / 文档

### 2. 分析内容
- 识别产品的核心功能
- 梳理用户使用产品的典型情境
- 分析不同场景下的用户行为

### 3. 输出标准化文档

---

## 输出模板

```markdown
# 使用场景文档

> 文档生成时间：YYYY-MM-DD
> 数据来源：[文件名]

---

## 1. 场景概述

简要描述产品的核心使用场景是什么。

## 2. 核心使用场景

### 场景 1：[场景名称]

| 项目 | 内容 |
|------|------|
| **场景描述** | [具体描述] |
| **触发条件** | [什么情况下会发生] |
| **用户目标** | [用户想达成什么] |
| **操作路径** | [主要步骤] |
| **预期结果** | [成功状态] |
| **使用频率** | 高 / 中 / 低 |

### 场景 2：[场景名称]
...

## 3. 场景矩阵

| 场景 | 用户类型 | 触发条件 | 频率 | 重要性 |
|------|----------|----------|------|--------|
| [场景1] | [用户群] | [条件] | 高 | P0 |
| [场景2] | [用户群] | [条件] | 中 | P1 |

## 4. 场景上下文

### 4.1 时间维度
- [ ] 碎片时间（几分钟）
- [ ] 专注时段（较长时间）
- [ ] 无明显时间特征

### 4.2 环境维度
- [ ] 安静环境
- [ ] 嘈杂环境
- [ ] 移动中
- [ ] 固定位置

### 4.3 设备维度
- [ ] 手机
- [ ] 平板
- [ ] 电脑

### 4.4 网络维度
- [ ] 需要稳定网络
- [ ] 可离线使用
- [ ] 弱网环境可用

## 5. 信息完整性检查

**缺失项**：[列出所有待补充项目]
```

---

## 执行要求

1. 读取用户提供的文档
2. 提取使用场景相关信息
3. 按模板输出文档到 `docs/product-design/[项目名]/使用场景.md`
4. 标注缺失信息并提示用户

---

## 示例

### 输入
```
用户: "帮我梳理下这个产品的使用场景"
文件: /design/ecommerce/index.html (电商设计稿)
```

### 输出
生成文件: `docs/product-design/ecommerce/使用场景.md`
```

### 流程
1. Claude 读取设计稿 HTML 文件
2. 识别主要页面和 功能模块
3. 提取用户操作流程
4. 按场景模板输出
```
