# 需求背景分析 Skill

基于用户提供的文档（HTML文件夹、压缩包、Word、PDF等），分析并输出标准化的需求背景文档。

## 输出路径

```
docs/product-design/[项目名]/需求背景.md
```

## 工作流程

### 1. 接收输入

用户可能提供以下形式的输入：
- HTML 文件夹（设计稿导出）
- HTML/ZIP 压缩包
- Word/PDF 文档
- 纯文本描述

### 2. 分析文档

阅读并提取以下信息：
- **业务背景**：为什么要做这个？解决什么问题？
- **用户信息**：目标用户是谁？他们的诉求是什么？
- **功能范围**：包含哪些功能？边界在哪里？
- **成功标准**：如何衡量这个项目是否成功？

### 3. 输出标准化文档

按照下方模板输出。如果信息缺失，标注 `[待补充]` 并提示用户。

## 执行要求

1. **先读取文档**：使用 Read 工具读取用户提供的文件
2. **提取信息**：从文档中提取相关信息填入模板
3. **标注缺失**：无法从文档中获取的信息，标注 `[待补充]`
4. **提示补充**：在文档末尾列出所有缺失项，提示用户补充
5. **输出位置**：将生成的文档保存到 `docs/product-design/[项目名]/需求背景.md`
